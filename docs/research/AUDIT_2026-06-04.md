## 4. AI Dictionary System

### Архитектура

```
aiDictionaryProvider.ts (клиент)
  ├─ isSimpleWord() → простая/сложная логика
  ├─ getAiKeys() → чтение API-ключей из settingsStore
  ├─ getAuthHeaders() → JWT токен из localStorage
  └─ fetch('/api/ai/dictionary') → серверный endpoint

api/ai/dictionary/route.ts (сервер)
  ├─ resolveModel():
  │   ├─ AI_GATEWAY_API_KEY (env var) → Vercel Gateway
  │   ├─ geminiApiKey (body) → @ai-sdk/google
  │   └─ deepseekApiKey (body) → @ai-sdk/openai-compatible
  ├─ simpleDefinition() → maxOutputTokens: 150
  └─ fullDefinition() → maxOutputTokens: 900, JSON-structured
```

### Что работает (проверено curl)
- ✅ Health-check: `/api/ai/dictionary` с `complexity: 'health'` — без авторизации
- ✅ С личным API-ключом: авторизация приложения не требуется
- ✅ Без ключа: требуется `validateUserAndToken`
- ✅ DeepSeek API достигается: возвращает "Authentication Fails" для фейкового ключа
- ✅ Gemini API достигается: возвращает "API key not valid" для фейкового ключа
- ✅ Модели подтверждены через документацию: `deepseek-v4-pro`, `gemini-2.5-flash-lite`

### Что требует отладки в браузере
- 🔴 TimeoutError при DOM-обновлении — возможно, бесконечный ререндер в DictionaryPopup
- 🔴 GeminiProvider: добавлен `getAIFetch()` для Tauri desktop (был пропущен)
